How the gaps were identified

1st gap

We started asking ChatGPT to find cultural properties related to Benozzo Gozzoli. Here is the question we made and the answer of the LLM:

Can you make a SPARQL query on ArCo which can ask the following question: what are the cultural properties of the author Benozzo Gozzoli?

ChatGPT’s answer:

We need to find cultural properties (typically instances of arco-core:CulturalProperty) that are linked to the author Benozzo Gozzoli as their creator.

Here's a SPARQL query that does that:

PREFIX arco-core: <https://w3id.org/arco/core/>

PREFIX arco-cd: <https://w3id.org/arco/ontology/context-description/>

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT DISTINCT ?culturalProperty ?title ?type

WHERE {

?culturalProperty a arco-core:CulturalProperty ;

arco-core:hasAgentRole ?agentRole .

?agentRole a arco-cd:Author ;

arco-core:hasAgent ?author .

?author rdfs:label "Benozzo Gozzoli"@it .

OPTIONAL { ?culturalProperty rdfs:label ?title . }

OPTIONAL { ?culturalProperty a ?type . }

}

We tried this query on the SPARQL endpoint of ArCo ( OpenLink Virtuoso SPARQL Query Editor ), it worked but it didn’t give any result. Therefore we said to the LLM:

There are no results for this query on ArCo.

ChatGPT’s answer:

Thanks for checking — you're right, the query returns no results  as written, likely because of one or more of the following issues:

  1. Benozzo Gozzoli may not be labeled exactly as "Benozzo Gozzoli"@it in the ArCo dataset.
  2. The author might be represented as a URI, not a literal label.
  3. The role or relationship pattern could differ, i.e., arco-core:hasAgentRole might not be the correct property path.

Let me refine the approach by searching for the actual URI of Benozzo Gozzoli and adjusting the structure accordingly.

2nd gap

We wanted to know the year of birth of Benozzo Gozzoli. We asked:

Can you write a SPARQL query to get the year of birth of Benozzo Gozzoli?

ChatGPT’s answer:

To get the year of birth of Benozzo Gozzoli, we need to query for a person (foaf:Person or similar class) and find their birth date. Here’s a general SPARQL query:

PREFIX foaf: <http://xmlns.com/foaf/0.1/>

PREFIX dbo: <http://dbpedia.org/ontology/>

SELECT ?birthDate

WHERE {

?person foaf:name "Benozzo Gozzoli"@en ;

dbo:birthDate ?birthDate .

}

This query should work on endpoints like DBpedia, provided they support the foaf:name and dbo:birthDate properties.

Our feedback:

We tried this query on DBpedia’s SPARQL endpoint and it worked! It returned “1420-01-01”^^xsd:date. So it is effective, but you need to know which SPARQL endpoint contains the data.

We also realized that it helps to try “foaf:name” vs “rdfs:label”, depending on the source.

3rd gap

We asked the following question, expecting results from the ArCo dataset:

Can you list all the artists in ArCo with their place of birth and date of birth?

ChatGPT’s answer:

Here’s a SPARQL query to retrieve artists in ArCo with their place and date of birth:

PREFIX arco-core: <https://w3id.org/arco/core/>

PREFIX arco-lite: <https://w3id.org/arco/ontology/arco-lite/>

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

PREFIX arco-dd: <https://w3id.org/arco/ontology/denotative-description/>

SELECT ?artist ?name ?birthPlace ?birthDate

WHERE {

?artist a arco-core:Agent ;

rdfs:label ?name ;

arco-dd:hasBirthPlace ?birthPlace ;

arco-dd:hasBirthDate ?birthDate .

}

Our feedback:

This query returned an error: apparently the properties used (e.g., hasBirthPlace) were incorrect. The endpoint didn’t recognize them.

We realized that we needed to explore the ontology of ArCo better, to find the actual properties used for biographical information. Eventually, we learned that:

  • arco-core:Agent is a superclass for people or organizations
  • arco-cd:Person is a subclass that represents individual persons
  • birth-related data is typically in arco-dd (denotative-description)
  • hasBirthPlace and hasBirthDate may not be present at all for many artists

So we adjusted our query and limited it to persons and just birth dates, yielding this improved version:

PREFIX arco-core: <https://w3id.org/arco/core/>

PREFIX arco-cd: <https://w3id.org/arco/ontology/context-description/>

PREFIX arco-dd: <https://w3id.org/arco/ontology/denotative-description/>

PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?person ?name ?birthDate

WHERE {

?person a arco-cd:Person ;

rdfs:label ?name .

OPTIONAL { ?person arco-dd:hasBirthDate ?birthDate . }

}

This query returned some results! It proved that smaller and focused queries help more than overly ambitious ones.

Conclusion

We have seen that using SPARQL queries on cultural heritage datasets like ArCo and DBpedia can be challenging due to the structure of the data and vocabulary mismatches. However, with patience and by exploring ontologies, useful results can be achieved.
